## üìÇ Directory Descriptions

### Root Level
- **Configuration Files**: Project configuration, dependencies, and entry points
- **Documentation**: README and detailed guides
- **PlantUML Files**: Diagram source files (reference templates)

### `src/` - Source Code

#### `src/components/` - Vue Components
- **`common/`**: Shared/reusable components used across the app
  - `ProjectPageTemplate.vue`: Wrapper template for all project pages
  - `ReusableCard.vue`: Card component with 3D header effect
  - `ContactForm.vue`: Contact form with Google Cloud Functions (SMTP) integration
  - `ContactCard.vue`: Contact information cards
  
- **`home/`**: Home page section components
  - `Hero.vue`: Hero section
  - `About.vue`: About section
  - `Services.vue`: Services grid/cards
  - `Portfolio.vue`: Portfolio projects grid
  - `Contact.vue`: Contact section wrapper
  
- **`layout/`**: Layout and navigation components
  - `Navigation.vue`: Main navigation sidebar
  - `NavButton.vue`: Navigation button component
  
- **`projects/`**: Project-specific components
  - `ProjectHeroCard.vue`: Project hero section
  - `ProjectGallery.vue`: Image gallery
  - `TechnologyStack.vue`: Technology stack visualization
  - `DiagramViewer.vue`: Interactive diagram viewer (with/without narration)
  - `ArchitectureOverview.vue`: Architecture layers overview
  - `EngineeringChallenges.vue`: Challenges and solutions
  - `PerformanceMetricsSection.vue`: Performance metrics and charts
  - And more...
  
- **`services/`**: Service page components
  - `ServiceHeroSection.vue`: Service hero section
  - `ServiceOverview.vue`: Service overview content
  - `ServiceCapabilities.vue`: Capabilities list
  - `ServiceProcess.vue`: Process steps
  - `ServiceCaseStudies.vue`: Case studies
  - `ServiceFAQ.vue`: FAQ section
  - `ServiceCTA.vue`: Call-to-action section

#### `src/views/` - Page Views (Routes)
- **`Home.vue`**: Main home page (includes all home sections)
- **`projects/`**: Individual project detail pages
  - Each project has its own page (e.g., `HeatExchangerPage.vue`)
  - Uses `ProjectPageTemplate` as wrapper
  - Contains project-specific data and components
- **`services/`**: Individual service detail pages
  - Each service has its own page (e.g., `FullStackDevelopmentPage.vue`)
  - Uses service components for content

#### `src/router/` - Routing
- **`index.js`**: Vue Router configuration
  - Defines all routes (`/`, `/projects/*`, `/services/*`)
  - Lazy loading for route components

#### `src/config/` - Configuration
- **`constants.js`**: Application constants
  - SMTP configuration (Google Cloud Functions)
  - Contact information
  - Technology categories
  - Project icon names
  - And more...
- **`*_Narration.js`**: Narration configurations (optional, for diagrams with narration)

#### `src/utils/` - Utilities
- **`iconResolver.js`**: Dynamic icon resolution utility
  - Resolves technology/service icons
  - Priority: Devicon > Local Icons > Emoji Fallback

#### `src/assets/` - Assets
- **`css/`**: Stylesheets
  - `main.css`: Main global stylesheet

### `public/` - Public Assets

#### `public/assets/img/` - Images
- **`Icons/`**: Technology and service icons (used by iconResolver)
- **`services/`**: Service page images (hero, process, CTA, banners)
- Other project images (gallery images, etc.)

#### `public/assets/diagrams/` - Diagrams
- Architecture diagram SVG files
- Used by `DiagramViewer` component
- Generated from PlantUML source files (`.puml` in `docs/diagrams/`)

#### `public/assets/vendor/` - Third-party Libraries
- Bootstrap CSS/JS
- AOS (Animate On Scroll)
- GLightbox
- Swiper
- And other vendor libraries

## üîÑ File Flow

### Creating a New Project Page
1. Create page in `src/views/projects/YourProjectPage.vue`
2. Add route in `src/router/index.js`
3. Use components from `src/components/projects/`
4. Add diagram SVG to `public/assets/diagrams/`
5. (Optional) Add narration config to `src/config/`

### Creating a New Service Page
1. Create page in `src/views/services/YourServicePage.vue`
2. Add route in `src/router/index.js`
3. Use components from `src/components/services/`
4. Add images to `public/assets/img/services/`
5. Update `src/components/home/Services.vue` to link to new page

### Icon Resolution
1. Technology name ‚Üí `src/utils/iconResolver.js`
2. Resolves to: Devicon CDN / Local icon / Emoji fallback
3. Local icons located in `public/assets/img/Icons/`

## üìù Important Notes

- **All `.puml` files** in `docs/diagrams/` are kept as reference templates for diagram creation
- **All `.svg` files** in `public/assets/diagrams/` are used by project pages
- **Documentation** in `docs/` folder provides detailed guides
- **Configuration** in `src/config/constants.js` centralizes app settings
- **Components** follow a clear hierarchy: common ‚Üí home/layout/projects/services
